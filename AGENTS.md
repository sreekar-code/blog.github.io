## Build / Lint / Test Commands

This is a **static HTML/CSS site** self-hosted on a Raspberry Pi, with GitHub as the middleman.

- **No build process**: Files are served directly as static assets
- **No linting**: No ESLint, Stylelint, or other linters configured
- **No tests**: No test framework configured
- **To preview locally**: `python3 -m http.server 8000` from the repo root
- **To publish**: commit + push to `main` в†’ GitHub Action runs в†’ `sudo git pull` on Pi

## Code Style Guidelines

### HTML

- Use HTML5 doctype: `<!DOCTYPE html>`
- Include viewport meta tag: `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- Use semantic tags: `<main>`, `<section>`, `<header>`, `<footer>`
- Indent with 4 spaces
- Use double quotes for attributes
- Analytics script (Umami) must be in `<head>`:
  ```html
  <script async src="https://analytics.balla.dev/script.js" data-website-id="2d661ca0-e926-4c3c-8d8c-e948998a7ac8"></script>
  ```
- CSS link must include cache-busting version: `<link rel="stylesheet" href="styles.css?v=47">`
- Blog posts in `blog/` use `../styles.css?v=47` and `../nav.js?v=3`
- Every HTML file must include `nav.js` before `</body>` (root pages use `nav.js?v=3`, blog posts use `../nav.js?v=3`)

### CSS

- All styles in `styles.css` (shared across all pages)
- **Theme:** Journal paper вЂ” cream page (`#f4ecce`, `--bg`) on a dark wood desk (`#1a0e06`, `--desk`)
- **Font:** Kalam (Google Fonts, weights 300, 400 + 700), loaded via `@import` at the top of `styles.css`
  - `font-family: 'Kalam', 'Apple Color Emoji', 'Segoe UI Emoji', 'Noto Color Emoji', cursive`
  - Emoji fonts are included in the stack so emojis render correctly without inheriting the text-shadow blur
- **`html` = desk**, `body` = paper page. Do not mix them up.
- **`--line-h`** must stay calibrated to `font-size-em Г— 16 Г— line-height(2)` so ruled lines align with text:
  - Desktop: `1.4em Г— 16 Г— 2 = 45px`
  - Mobile: `1.25em Г— 16 Г— 2 = 40px`
  - Recalculate and update `--line-h` whenever font-size changes
- Mobile styles use `@media (max-width: 768px)`; desktop layout overrides use `@media (min-width: 600px)`
- CSS custom properties used throughout for theming (see `:root` block)
- No CSS frameworks (Tailwind, Bootstrap, etc.)
- When updating `styles.css`, bump the version in **all** HTML files:
  ```bash
  find . -name "*.html" -exec sed -i '' 's/styles\.css?v=N/styles.css?v=N+1/g' {} +
  ```
  Replace `N` and `N+1` with the current and next version numbers.

### JavaScript

- Vanilla JavaScript only (no frameworks)
- Use `const` and `let` (avoid `var`)
- Function declarations preferred over arrow functions for top-level functions
- Use `localStorage` for client-side persistence

### nav.js вЂ” Mobile Navigation

- `nav.js` is an IIFE that runs only on mobile (`window.innerWidth <= 768`)
- Hides the inline nav `<p>` and injects a hamburger button + slide-in sidebar
- On blog post pages (detected by presence of `<h1>`), also hides `.blogtitle` and the `<hr>` below the nav
- When `nav.js` is modified, bump its version in **all** HTML files:
  ```bash
  find . -name "*.html" -exec sed -i '' 's/nav\.js?v=N/nav.js?v=N+1/g' {} +
  ```

### Flow Page Notes

- `flow.html` shows a GitHub-style contribution graph: full year, week columns (MonвЂ“Sun rows)
- Year navigation: в†ђ prev / в†’ next buttons; next button hidden when viewing current year

### Publishing a New Post

1. Create the HTML file in `blog/` using the template above
2. Add it to the top of the `<ul>` in `blog.html`
3. Commit and push вЂ” GitHub Action auto-updates `feed.xml` and `dates.json`
4. `sudo git pull` on the Pi

### Naming Conventions

- **HTML files**: ASCII only, Title Case with hyphens (e.g., `What-Is-A-Problem.html`). **No special characters, em dashes, or non-ASCII characters in filenames** вЂ” Nginx cannot serve them reliably.
- **CSS classes**: lowercase, hyphenated (e.g., `.blogtitle`, `.site-footer`)
- **Images**: lowercase, hyphenated (e.g., `myway.jpg`, `gokarna.jpg`)

### Images

- Wrap standalone images in a `<p>` tag: `<p><img src="..." alt="..."/></p>`
- The CSS `p:has(> img)` selector styles the `<p>` as a photo frame with scrapbook tape corners
- Images get: cream border, sepia filter, slight tilt, warm drop shadow
- Do not add classes or inline styles to image `<p>` wrappers вЂ” CSS handles everything

### Blockquotes

- Rendered as sticky notes: warm aged-yellow background (`#ede8b4`), slight tilt, warm drop shadow
- Use `<blockquote><p>...</p></blockquote>` markup as normal вЂ” CSS handles the look

### Footer

- The `<hr>` + `<footer class="site-footer">` block (Source + RSS links) appears **only on `index.html`**
- Do not add it to Flow, Cooking, Blog, or individual post pages

### Links

- Internal links: relative paths (e.g., `blog.html`, `../index.html`)
- External links: use `target='_blank'` attribute
- Navigation uses `&nbsp;&nbsp;` for spacing between links

### Git Conventions

- Commit messages should describe the change concisely
- Push to `main` branch only
